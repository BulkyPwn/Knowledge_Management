# AI 一站式桌面 — 工具使用指南

## 1. 概述

AI 一站式桌面集成了 **文档设计、代码开发、问题定位、知识管理、Agent 对话** 五大 AI 辅助工作模式，为开发者提供一站式 AI 辅助工作台。

## 2. 启动应用

### 2.1 直接运行

双击安装目录下的 `AI_Assistant.exe` 启动应用。

### 2.2 启动后端服务（可选）

若需要使用代码开发的完整文件浏览功能和知识管理功能，需额外启动后端服务。请联系管理员获取后端服务的启动方式。

## 3. 功能模块详解

### 3.1 一键设计（Document Design）

用于快速生成项目设计文档。

**使用步骤：**

1. 在左侧导航栏点击「一键设计」。
2. 选择目标项目或模块。
3. 选择参考文档（可选）。
4. 指定设计文档的工作路径（输出目录）。
5. 点击「生成」，AI 将自动生成设计文档。
6. 可在历史记录中查看之前生成的文档。

### 3.2 代码开发（Code Development）

集成代码编辑器、文件浏览器和 AI 对话的全功能开发环境。

**主要功能：**

- **代码编辑器**：提供语法高亮、代码补全、多标签页的编辑体验。
- **文件浏览器**：浏览本地文件系统，支持打开和编辑文件。
- **代码问答面板**：选中代码后向 AI 提问，获取代码解释、优化建议等。
- **内置终端**：可在应用内直接执行命令行操作。
- **命令面板**：快速访问各项功能。
- **Problems / Output 面板**：查看代码问题和输出信息。
- **状态栏**：显示当前文件、光标位置等信息。

**键盘快捷键：**

| 快捷键 | 功能 |
|---|---|
| `Ctrl + P` | 打开命令面板 |
| `Ctrl + Shift + `` ` `` | 切换终端面板 |
| `Ctrl + Shift + I` | 打开开发者工具（调试用） |

### 3.3 问题定位（Issue Location）

上传日志和代码文件，AI 辅助分析和定位问题。

**使用步骤：**

1. 在左侧导航栏点击「问题定位」。
2. 上传相关日志文件（支持 `.log`、`.txt` 等格式）。
3. 上传相关代码文件。
4. 选择分析工具类型：
   - **日志解析**：提取日志中的关键信息和异常。
   - **代码分析**：对代码进行静态分析。
   - **依赖检查**：检查项目依赖关系和潜在冲突。
   - **性能分析**：分析性能瓶颈和优化建议。
5. 点击「分析」，AI 将输出分类后的分析结果。

### 3.4 知识管理（Knowledge Management）

创建和管理知识库，支持基于知识库内容的智能问答。

**使用步骤：**

1. 在左侧导航栏点击「知识管理」。
2. **创建知识库**：点击新建，输入知识库名称。
3. **导入文件**：选择知识库，导入本地文件或文件夹。
4. **管理文件列表**：查看、删除知识库中的文件。
5. **选择模型**：支持 MiniMax、GLM-5、GLM-4 及自定义模型配置。
6. **知识库问答**：在对话框中提问，AI 将基于知识库内容进行智能回答。
7. **图片上传**：支持上传图片到知识库。

### 3.5 Agent

通用 AI 对话功能，支持多种 Agent 角色。

**支持的角色：**

| 角色 | 说明 |
|---|---|
| 灵犀 | 通用 AI 助手，回答各类问题 |
| CodeAgent | 专注于代码编写、调试、优化的编程助手 |
| 创意助手 | 提供创意灵感和内容生成 |
| 专家模式 | 深度技术问题解答和架构讨论 |

**使用步骤：**

1. 在左侧导航栏点击「Agent」。
2. 在顶部切换栏选择 Agent 角色。
3. 在输入框中输入问题。
4. AI 将根据所选角色返回相应的回答。

## 4. 设置面板

点击左侧导航栏底部的「设置」图标进入。

| 设置项 | 说明 |
|---|---|
| 主题切换 | 深色（dark）/ 浅色（light）/ 灰色（gray）三种主题 |
| 后端地址 | 配置后端服务的地址（默认 `http://localhost:5000`） |
| 模型选择 | 选择不同 AI 模型 |
| 代理配置 | 配置网络代理地址 |

## 5. 常见问题

### Q: 代码开发模块中文件树为空？

请确认后端服务已正常启动。若未启动后端，应用将使用内置默认文件树。

### Q: 如何切换 AI 模型？

在「设置」面板中可选择不同模型。知识管理模块支持独立选择模型。

### Q: 知识管理回答不准确？

请检查知识库中是否已导入相关文件，以及所选模型是否正确。

### Q: 如何反馈问题或寻求帮助？

请联系系统管理员或项目维护团队。
