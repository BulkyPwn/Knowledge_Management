# 内容总结技能

当用户需要对内容进行总结时，按以下流程执行：

## 步骤

### 场景 A：总结知识库内容

1. 调用 `knowledge_query` 搜索相关文档
2. 阅读检索到的内容
3. 生成结构化总结：
   - **核心要点**（3-5 条）
   - **详细内容摘要**
   - **关键术语和定义**（如有）
   - **相关主题**（建议进一步阅读）

### 场景 B：总结网页内容

1. 调用 `fetch_url` 获取网页内容
2. 分析文章结构
3. 生成结构化总结：
   - **一句话概括**
   - **核心论点**
   - **关键数据和事实**
   - **作者观点/立场**
   - **来源 URL**

### 场景 C：对比总结

当用户要求对比多个来源时：
1. 分别获取各来源内容
2. 提取各自的要点
3. 制作对比表格或列表
4. 总结共同点和差异

## 注意事项

- 保持客观，不添加原文没有的信息
- 标注信息来自哪个来源
- 对于长篇内容，先给出概览，再提供详细总结
- 保留重要的数据和引用
